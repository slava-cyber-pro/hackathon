How we built it

CLAUDE.md & Build Process

# CLAUDE.md — key rules ## Code Principles - Small functions: max ~20 lines - One responsibility per file - No business logic in routes - Type everything: strict TS + Python hints - Fail explicitly, never swallow errors ## Backend - Async everything: routes, DB, Redis - Stateless JWT auth (no token storage) - Domain exceptions, not HTTPException - Prometheus + structlog observability ## Frontend - TanStack Query for server state - Zustand for client state - shadcn/ui + Tailwind CSS - React Hook Form + Zod validation ## Testing - Real DB, no mocks (backend) - MSW for network mocking (frontend) - Playwright for E2E - Minimum 80% coverage enforced
